# 継続作業ルール

このリポジトリでは、別セッションでも同じ流れで作業を継続する。

## 基本方針

- まとまったタスクごとに `codex/<task-name>` ブランチを作成する。
- 実装、検証、ドキュメント更新を同じタスクの完了条件に含める。
- `main` へ直接コミットしない。
- PR はタスクごとに作成し、チェックとレビュー観点を確認してからマージする。
- マージ後、未完了タスクが残っていれば `main` を更新して次のブランチを作る。
- 作業状態は `docs/TASKS.md` に更新する。

## 1 タスクの手順

1. `git status --short --branch` で作業状態を確認する。
2. `main` を最新化する。
3. `codex/<task-name>` ブランチを作成する。
4. タスクの実装と必要なドキュメント更新を行う。
5. `npm test` とブラウザでの動作確認を行う。
6. 変更範囲を確認してコミットする。
7. ブランチを push し、PR を作成する。
8. PR をマージする。
9. `docs/TASKS.md` の完了状態を更新する。

## 公開

- GitHub Pages は `.github/workflows/pages.yml` で `main` への push 時にデプロイする。
- 公開 URL は `https://unkei.github.io/ai-rtype2/` を想定する。
- Pages の初回有効化が未設定の場合は、GitHub Actions を Pages の source として設定する。

## ゲーム要件

- R-Type ライクな横スクロールシューティングであること。
- キーボード、GamePad、スマホのタッチ操作に対応すること。
- 画面サイズに追従し、モバイルでタッチ UI を表示すること。
- ブラウザのみで起動でき、ビルド手順なしで Pages に載せられること。
